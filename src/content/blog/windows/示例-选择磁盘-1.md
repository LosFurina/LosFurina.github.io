删除 EFI 系统分区 (ESP) 是一项高风险操作，通常仅适用于非系统盘（例如，旧的启动盘现在用作数据盘）或您打算完全重装系统的磁盘。请谨慎操作。

> ⚠️ **严重警告：**
>
> *   **绝对不要删除您当前 Windows 系统所在硬盘的 EFI 分区**。这将导致您的计算机无法启动。
> *   此操作会永久删除分区及其中的数据。在开始之前，请务必备份磁盘上的所有重要资料。
> *   在操作过程中，请再三确认您选择的磁盘和分区编号，避免误删数据。

Windows 内置的“磁盘管理”工具通常不允许直接删除 EFI 分区（“删除卷”选项为灰色）。 因此，我们需要使用命令行工具 **Diskpart** 来强制执行此操作。

## 方法一：使用 Diskpart 精确删除 EFI 分区

此方法仅删除指定的 EFI 分区，保留磁盘上的其他分区。

1.  **以管理员身份运行命令提示符**
    *   在任务栏搜索框中输入 `cmd`。
    *   在搜索结果中右键点击“命令提示符”，选择“以管理员身份运行”。

2.  **启动 Diskpart 工具**
    *   在打开的黑色命令提示符窗口中，输入以下命令并按回车：
      ```bash
      diskpart
      ```

3.  **列出并选择目标磁盘**
    *   输入 `list disk` 列出所有已连接的磁盘。根据磁盘大小识别您要操作的磁盘。
    *   使用 `select disk X` 命令选择目标磁盘 (将 `X` 替换为目标磁盘的编号)。
      ```bash
      # 示例：选择磁盘 1
      select disk 1
      ```

4.  **列出并选择 EFI 分区**
    *   输入 `list partition` 列出所选磁盘上的所有分区。
    *   在列表中找到类型为“系统”的分区，并记下其分区编号。
    *   使用 `select partition Y` 命令选择 EFI 分区 (将 `Y` 替换为 EFI 分区的编号)。
      ```bash
      # 示例：选择分区 1
      select partition 1
      ```

5.  **强制删除分区**
    *   输入以下命令以强制删除受保护的 EFI 分区。`override` 参数是此操作成功的关键。
      ```bash
      delete partition override
      ```
    *   操作成功后，输入 `exit` 退出 Diskpart。

## 方法二：使用 `clean` 命令清除整个磁盘

如果您希望删除目标磁盘上的**所有分区**（包括 EFI 分区）并将其完全重置，`clean` 命令是更快捷的选择。

> ⚠️ **警告：** 此命令会立即并永久地清除所选磁盘上的**全部数据和分区表**，请务必确认选择了正确的磁盘。

1.  **以管理员身份打开 `diskpart`** (步骤同上)。
2.  **列出并选择目标磁盘** (步骤同上)。
    ```bash
    diskpart
    list disk
    # 再次确认选择了正确的磁盘！
    select disk X 
    ```
3.  **执行清除命令**
    *   输入以下命令：
      ```bash
      clean
      ```
    *   该命令会快速清除磁盘。完成后输入 `exit` 退出。

## 重新分配磁盘空间

删除 EFI 分区或清理整个磁盘后，该空间会变为“未分配”状态。您现在可以使用图形化的“磁盘管理”工具来重新利用这部分空间。

1.  **打开磁盘管理**
    *   右键点击“开始”按钮，选择“磁盘管理”。

2.  **初始化磁盘 (如果使用了 `clean` 命令)**
    *   如果使用了 `clean` 命令，磁盘管理会提示您初始化磁盘。
    *   选择 **GPT** (适用于大于2TB的磁盘和现代系统) 或 MBR，然后点击“确定”。

3.  **创建新分区或扩展现有分区**
    *   在磁盘管理中，您会看到一块黑色的“未分配”空间。
    *   **创建新分区**：右键点击该“未分配”空间，选择“新建简单卷”，然后按照向导提示设置分区大小、驱动器号和格式。
    *   **扩展现有分区**：如果未分配空间紧邻您想扩展的分区右侧，您可以右键点击该分区，选择“扩展卷”，然后按照向导将其合并。
